# Accessibility Guidelines (A11Y): Optima

Optima se compromete a ser una herramienta inclusiva, siguiendo los estándares **WCAG 2.1 nivel AA**.

## 1. Contraste y color
- **Ratios de contraste:** Todo texto debe mantener un ratio mínimo de **4.5:1** contra su fondo (**3:1** para texto grande o componentes de interfaz).
- **No dependencia del color:** Los estados (error, éxito, advertencia) no deben comunicarse solo mediante el color. Deben ir acompañados de íconos o etiquetas de texto claras.
  - Ejemplo: Una licencia próxima a vencer usa color rojo y un ícono de advertencia '⚠️'.

## 2. Navegación por teclado
- **Orden de Foco:** El orden de tabulación debe ser lógico y predecible (de arriba hacia abajo, de izquierda a derecha).
- **Indicador de foco:** Nunca eliminar el `outline` de foco. Usar el estilo de Focus Ring de **shadcn/ui** (color `Accent` con contraste suficiente).
- **Saltar al contenido:** Implementar un link oculto ("Skip to content") para usuarios de lectores de pantalla.

## 3. Lectores de pantalla (ARIA)
- **Etiquetas Alt:** Todas las imágenes decorativas deben tener `alt=""`. Gráficos y botones con íconos deben tener `aria-label` descriptivos.
- **Roles Semánticos:** Usar HTML semántico (`<nav>`, `<main>`, `<article>`, `<aside>`) en lugar de `<div>` genéricos.
- **Live Regions:** Usar `aria-live="polite"` para notificaciones de Toast o actualizaciones dinámicas del dashboard que no requieren interrupción inmediata.

## 4. Tipografía y escalabilidad
- **Tamaño Mínimo:** 16px para cuerpo de texto.
- **Unidades Relativas:** Usar `rem` y `em` en lugar de `px` para permitir que el zoom del navegador funcione correctamente sin romper el layout.

## 5. Herramientas de auditoría
- **Lighthouse:** Ejecución mensual de auditorías con meta de puntuación > 90.
- **Axe DevTools:** Pruebas continuas en componentes críticos durante el desarrollo.

---
**Justificación:** El cumplimiento de A11Y no es opcional. Un sistema financiero debe ser operable por cualquier profesional, independientemente de sus capacidades visuales o motoras.
